# 修正后的产品理解

更新时间：2026-05-20

## 核心定位

**EnvForge**（原名 Fool Server Restore）是一个自托管的虚拟机/服务器配置管理网站，不是以 Git/GitHub 为中心的配置仓库管理工具。

服务器运行后只是服务提供者，提供 Web UI 和 API。用户可以在网站中登录，然后通过 SSH、WinRM、Docker context 或其他连接方式连接当前电脑、服务器或目标虚拟机，对目标机器上的软硬件信息、系统配置、应用软件进行查看、选择、上传、组合和应用。

## 用户能做什么

- 以游客身份浏览基础能力。
- 在页面内注册、登录、登出和管理个人信息。
- 管理自己上传过的虚拟机配置、软件清单和系统配置清单。
- 通过 SSH 等连接方式添加目标虚拟机。
- 查看当前连接虚拟机的硬件、系统、软件和配置详情。
- 自主选择要采集、上传、保存、应用的具体信息，而不是整机全部同步。
- 在“系统软硬件信息 add”页面搜索并选择可加入当前虚拟机的配置包、软件包、运行时、服务模板或系统策略。
- 从其他用户或自己的配置资料中选择某些配置，整合到当前连接的虚拟机。
- 上传当前虚拟机的软件、硬件、系统配置资料，形成自己的配置资产。
- 在中文和英文 UI 之间切换。

## 当前不再作为核心目标

- 不需要实现 Git 功能来管理各种配置信息。
- 不把 GitHub 同步作为主流程。
- 不把所有信息塞到一个页面。
- 不把恢复流程设计成只有命令行 dry-run 的 DevOps 面板。

## 主界面信息架构

左侧标签栏只保留最主要功能：

1. 当前虚拟机软硬件信息管理
2. 系统软硬件信息 add
3. 我的

页面内容应根据左侧标签切换，而不是所有模块平铺在一个页面。

## 页面要求

### 当前虚拟机软硬件信息管理

- 展示当前连接的虚拟机。
- 支持通过 SSH 等方式添加/连接目标虚拟机。
- 展示详细硬件信息：CPU、内存、磁盘、系统、架构、网络。
- 展示详细软件信息：运行时、系统软件、包管理器、服务、容器、开发工具。
- 展示详细配置清单：环境变量、Shell、服务、网络、防火墙、安全策略。
- 每一类信息都应有复选框，让用户选择是否采集、上传、保存或应用。

### 系统软硬件信息 add

- 点击该栏目时，顶部出现搜索框。
- 下方展示可加入当前虚拟机的配置/软件/系统策略卡片。
- 卡片参考 Chrome 应用商店式布局：图片区域、名称、评分/热度、简短说明、分类、加入按钮。
- 支持分类筛选，例如运行时、开发工具、数据库、容器、安全、网络、系统服务。

### 我的

- 默认为游客状态。
- 支持登录、注册、登出。
- 支持个人信息管理。
- 支持查看和管理自己上传的虚拟机配置资料。
- 支持上传当前虚拟机配置、软件信息。

## 语言切换

系统必须支持中文和英文切换。当前阶段先在前端实现语言切换，后续扩展到后端消息和持久化用户偏好。

## 安全原则

- SSH 私钥、密码、token 不得明文保存。
- 连接目标虚拟机时应明确连接方式和权限。
- 应用配置到虚拟机前必须展示影响范围。
- 用户可以细粒度选择要采集、上传、应用的信息。
- 危险操作必须二次确认。
