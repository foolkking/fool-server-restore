# 3x-ui 面板

3x-ui 是基于 Xray-core 的多协议代理面板。EnvForge 自动跑官方 `install.sh`，再用
`x-ui setting` 把端口、用户名、密码改成你填写的值，这样安装完成后立刻能用确定的
凭据登录。

## 安装内容

- ✅ Xray-core + 3x-ui 面板（来自 GitHub 官方源）
- ✅ systemd 单元 `x-ui.service`（已设开机自启）
- ✅ 端口 / 访问路径 / 管理员账号按你的填写值生效（**不再使用默认账号**）

## 表单字段说明

### 面板端口 `panel_port`

3x-ui 默认 54321。强烈建议改成一个不常见的高位端口（例如随机的 30000-60000 区间），
让自动扫描器找不到你的面板。

> 修改端口后记得在防火墙开放它：`sudo firewall-cmd --add-port={panel_port}/tcp --permanent`。

### 面板访问路径 `panel_path`

默认 `/`，也就是 `http://server-ip:port/` 直接是面板。生产环境建议改成 `/admin`、
`/x-y-z`、`/your-secret-path` 等，扫描器一般只扫根路径。**必须以 `/` 开头。**

### 管理员用户名 `admin_username`

不要用 `admin`、`root`、`xui`，这些是爆破字典的常客。建议用 6-12 位的随机字母数字组合
（比如你用过的废弃用户名）。

### 管理员密码 `admin_password`

**留空** EnvForge 会自动生成 24 位强密码（运行结束的日志末尾会清楚地显示出来一次，
请立刻复制保存）。也可以自己填，不强制长度，但建议至少 16 位混合字符。

## 后续

### 安全加固（强烈建议）

1. 在面板的"面板设置"里再次修改密码（双重确认你能登录）。
2. 用 EnvForge 的 `Certbot SSL` Playbook 给面板加上 HTTPS。
3. 在面板里启用 IP 白名单或 Telegram bot 二次验证。

### 常用命令

```bash
sudo x-ui                 # 进入交互式管理菜单（重启、改密、查看日志等）
sudo systemctl status x-ui
sudo journalctl -u x-ui -n 50
```

### 卸载

```bash
sudo x-ui uninstall   # 官方提供的卸载流程
```

## 隐私说明

- 用户名和密码会写到 EnvForge 的运行日志里（在终端面板可见）。任务历史也会保留这些值。
  如果不希望泄露，请在运行后到任务历史里删除该任务记录。
- 自动生成的密码只在运行的那一次显示，**不会**保存到 EnvForge 数据库；忘记的话只能用
  `sudo x-ui` 命令在服务器上重置。
